# 여기에만 — 네이버 플레이스 연동 & 배포 가이드

> 비개발자도 따라할 수 있는 단계별 가이드입니다.
> 총 소요 시간: 약 20~30분


## 📁 프로젝트 파일 구성

```
yeogiemanpwa/
├── index.html        ← 앱 전체 코드 (UI + 기능)
├── api/
│   └── search.js     ← 네이버 검색 API 서버 함수
├── manifest.json     ← PWA 앱 설정
├── sw.js             ← 오프라인 지원
├── vercel.json       ← Vercel 서버 설정
├── icon-192.png      ← 앱 아이콘
└── icon-512.png      ← 앱 아이콘 (큰 크기)
```


## 🔑 1단계: 네이버 API 키 발급 (10분)

### A. 네이버 검색 API 키 (맛집 검색용)

1. **https://developers.naver.com** 에 접속
2. 네이버 아이디로 **로그인**
3. 상단 메뉴에서 **"Application" → "애플리케이션 등록"** 클릭
4. 아래 정보 입력:
   - **애플리케이션 이름**: `여기에만`
   - **사용 API**: `검색` 체크
   - **환경 추가**: `WEB 설정` 선택 → 웹 서비스 URL에 `http://localhost:3000` 입력
     (배포 후 실제 URL도 추가)
5. **"등록하기"** 클릭
6. 발급된 **Client ID**와 **Client Secret**을 메모해둡니다

   ⚠️ Client Secret은 절대 외부에 공개하지 마세요!

### B. 네이버 지도 API 키 (지도 표시용, 선택사항)

1. **https://console.ncloud.com** 에 접속
2. 네이버 클라우드 **회원가입 및 로그인**
3. 콘솔 → **"AI·NAVER API" → "Application 등록"**
4. 아래 정보 입력:
   - **Application 이름**: `여기에만`
   - **Service 선택**: `Maps` → `Web Dynamic Map` 체크
   - **서비스 환경**: `Web 서비스 URL` 등록
5. 발급된 **Client ID**를 메모 (index.html에서 사용)


## 🚀 2단계: Vercel에 배포하기 (10분)

### GitHub 계정 만들기 (없는 경우)
1. **https://github.com** 에서 **Sign up**
2. 이메일, 비밀번호 설정 후 가입 완료

### Vercel 배포

1. **https://vercel.com** 에 접속
2. **"Sign Up"** → **GitHub 계정**으로 로그인
3. **"Add New" → "Project"** 클릭
4. **"Import Git Repository"** 아래의 **"Upload"** 클릭
   - 또는: 화면에 `yeogiemanpwa` 폴더를 **드래그 앤 드롭**
5. **"Deploy"** 클릭 → 배포 시작!

### ⚠️ 중요: 환경변수 설정 (API 키 입력)

배포 완료 후:

1. Vercel 대시보드에서 프로젝트 클릭
2. **"Settings"** 탭 → **"Environment Variables"** 클릭
3. 아래 두 개를 추가:

| Key | Value |
|-----|-------|
| `NAVER_CLIENT_ID` | 1단계 A에서 발급받은 Client ID |
| `NAVER_CLIENT_SECRET` | 1단계 A에서 발급받은 Client Secret |

4. **"Save"** 클릭
5. **"Deployments"** 탭 → 최신 배포 옆 **⋮ → Redeploy** 클릭

   ✅ 이제 네이버 플레이스 실시간 검색이 작동합니다!

### 네이버 지도 활성화 (선택)

index.html 파일의 상단에서 아래 주석을 찾아 해제하고,
`YOUR_NCP_CLIENT_ID`를 1단계 B에서 발급받은 키로 교체:

```html
<!-- 이 줄의 주석을 해제하세요 -->
<script src="https://oapi.map.naver.com/openapi/v3/maps.js?ncpClientId=여기에_키입력"></script>
```


## 📱 3단계: 앱처럼 설치하기

배포된 URL을 스마트폰에서 열고:

### 아이폰 (Safari)
1. Safari에서 배포 URL 열기
2. 하단 **공유 버튼** (□↑) 탭
3. **"홈 화면에 추가"** 선택

### 안드로이드 (Chrome)
1. Chrome에서 배포 URL 열기
2. 상단에 뜨는 **"홈 화면에 추가"** 배너 클릭
3. 또는 ⋮ 메뉴 → **"앱 설치"**


## 🔍 작동 구조 설명

```
사용자가 "비빔밥" 검색
       ↓
브라우저 → /api/search?query=전주 비빔밥  (Vercel 서버 함수 호출)
       ↓
서버 함수 → 네이버 검색 API 호출 (API 키 사용)
       ↓
네이버 → 실제 검색 결과 반환 (가게명, 주소, 카테고리, 전화번호 등)
       ↓
서버 함수 → 프랜차이즈 자동 필터링 → 결과를 브라우저에 전달
       ↓
브라우저 → 검색 결과 표시 → "여기에만에 등록" 버튼 → 하트 가능!
```


## 🔧 네이버 개발자 URL 등록 업데이트

배포가 완료되면 Vercel에서 발급된 URL(예: `https://yeogiemanpwa.vercel.app`)을
네이버 개발자 센터에도 등록해야 합니다:

1. **https://developers.naver.com** → 내 애플리케이션 → `여기에만`
2. **API 설정** → **WEB 서비스 URL**에 배포 URL 추가
3. 저장


## ❓ 문제 해결

### "API 키가 설정되지 않았습니다" 에러
→ Vercel 환경변수에 `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`이 제대로 입력됐는지 확인
→ 환경변수 설정 후 반드시 **Redeploy** 해야 적용됩니다

### 검색은 되는데 결과가 없음
→ 네이버 검색 API가 "전주 + 검색어"로 검색합니다. "전주 비빔밥" 같이 구체적으로 검색해보세요

### 네이버 지도가 안 보임
→ index.html 상단의 네이버 지도 스크립트 주석을 해제했는지 확인
→ NCP Client ID가 올바른지 확인
→ 네이버 클라우드에서 서비스 URL을 등록했는지 확인

### 데이터가 새로고침하면 사라짐
→ 현재 버전은 LocalStorage로 데이터를 저장합니다 (같은 기기에서는 유지)
→ 여러 사용자가 공유하려면 Supabase 등 데이터베이스 연동이 필요합니다
